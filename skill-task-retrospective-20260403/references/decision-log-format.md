# 决策日志格式说明

## 完整模板

```markdown
## [HH:MM] 决策点 #N
**问题**: <一句话描述要解决的核心问题>
**背景**: <相关上下文，可选>
**选项**:
- A: <方案> → <预期结果>
- B: <方案> → <预期结果>
- C: <方案> → <预期结果>
**选择**: <X>
**理由**: <核心判断依据，1-2句话>
**风险**: <已知风险或不确定性>
**预期结果**: <预期 outcome>
**验证方式**: <如何确认选择是正确的>
```

## 简化模板（快速记录）

```markdown
## [HH:MM] 决策 #N
**选**: <方案>
**因**: <理由>
**风险**: <风险>
```

## 场景分类 Tag

在决策标题后加标签：
- `[架构]` — 技术架构决策
- `[工具]` — 工具/库选型
- `[流程]` — 流程/方法选择
- `[设计]` — UI/UX 设计决策
- `[策略]` — 解决思路选择

示例：
```
## [10:30] 决策点 #1 [工具]
**问题**: 日志分析用 grep 还是专用工具？
```

## 决策质量评级

事后回顾时给决策打分：

| 等级 | 说明 |
|------|------|
| ⭐⭐⭐⭐⭐ | 信息充分，结果完美 |
| ⭐⭐⭐⭐ | 信息充分，结果良好 |
| ⭐⭐⭐ | 信息一般，结果可接受 |
| ⭐⭐ | 信息不足，结果有问题 |
| ⭐ | 决策失误，需要改进 |

在复盘时添加：
```markdown
**决策质量**: ⭐⭐⭐⭐（事后验证）
**教训**: <如果有>
```
