# Code Reviewer

**用途:** 对代码进行深度 review，输出结构化的问题报告和改进建议。

**触发词:** review、代码审查、代码 review、帮我看代码、代码有问题

**适用场景:**
- 用户粘贴代码让你 review
- 用户让你 review 某个文件
- 代码有 bug 需要定位
- 代码质量评估

## 工作流程

1. 接收代码（粘贴或文件路径）
2. 按维度分析：`正确性 | 安全性 | 性能 | 可读性 | 设计 | 测试`
3. 输出三部分：
   - 🔴 **严重问题**（必须修复）
   - 🟡 **优化建议**（推荐改进）
   - ✅ **亮点**（值得保留的做法）
4. 给出改进后的参考代码（针对严重问题）

## 输出格式

```
## 📋 Review 报告 — [语言/框架]

### 🔴 严重问题
| 位置 | 问题 | 建议 |
|------|------|------|
| main L12 | SQL 拼接存在注入风险 | 使用参数化查询 |

### 🟡 优化建议
...

### ✅ 亮点
...

### 💡 改进参考
\`\`\`[语言]
# 修复后的代码
\`\`\`
```

## 审查维度

| 维度 | 检查点 |
|------|--------|
| 正确性 | 逻辑错误、边界条件、空指针、类型错误 |
| 安全性 | 注入、认证、加密、敏感信息泄露 |
| 性能 | 时间复杂度、内存泄漏、数据库查询、N+1 |
| 可读性 | 命名、注释、函数长度、复杂度 |
| 设计 | 单一职责、开闭原则、依赖注入 |
| 测试 | 覆盖率、边界用例、mock 合理性 |

## 限制

- 不运行未标记为安全的代码
- 涉及文件操作需用户确认
- 外部 API 调用需用户授权
- 生成的改进代码仅供参考，需自行验证

## 示例触发

```
"帮我 review 这段 Python 代码"
"看看这个 JS 文件有什么问题"
"review: [粘贴代码]"
```
