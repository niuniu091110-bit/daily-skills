# AI Hallucination Detector（AI幻觉检测与事实核查）

## Overview

AI幻觉是指大模型生成听起来合理但实际上是错误、虚假或虚构的内容。本技能帮助用户在关键时刻识别幻觉，并提供实用的核查和grounding策略。

## When to Use This Skill

- 用户询问AI生成内容的可信度
- 帮用户核实AI给出的事实、数据、引用
- AI的回答涉及专业领域（医疗、法律、技术）需要验证
- 用户在做研究，AI提供了来源或数据
- 用户对AI的回答表示怀疑，想验证
- AI使用了绝对化语言（"所有研究表明..."、"绝对不会..."）
- AI给出了具体数字、日期、统计数据

## Hallucination Pattern识别

### 高风险信号（最可能幻觉）

```
❌ "根据xxx研究表明..."（未提供具体论文）
❌ "xxx公司发言人表示..."（无具体来源）
❌ 具体的数字统计（年份、人数、百分比）
❌ 具体的产品名称+功能组合
❌ 法律/医疗领域的确切条文或建议
❌ "所有人都知道..." / "绝对不会..."（绝对化语言）
❌ 嵌套引用（A说B说C说D说...）
```

### 中风险信号

```
⚠️ 使用模糊的时间词（"最近"、"不久前"）
⚠️ 对复杂问题的简单直接答案
⚠️ 与常见知识高度一致但无来源
⚠️ 语气非常自信但未引用
```

### 低风险信号

```
✅ 明确说明了信息来源（URL、论文链接、官方文档）
✅ 主动说"我不确定"、"需要验证"
✅ 承认知识截止日期
✅ 区分事实和推测
```

## 核查四步法

### Step 1: 溯源（Source Tracing）

要求AI提供信息来源，然后验证：

```markdown
请帮我核实以下信息的来源：
"[AI的原话]"

具体请检查：
1. 来源是否真实存在？
2. 来源是否权威？（官方/学术/可信媒体）
3. 信息是否被正确引用/解读？
```

### Step 2: 交叉验证（Cross-Reference）

用多个独立来源验证：

```markdown
我需要验证以下说法，请用至少2个独立来源核实：
"[待验证内容]"

检查要点：
- 不同来源是否一致？
- 是否有来源相互矛盾？
```

### Step 3: 逻辑检验（Logic Check）

检查推理链是否有跳跃：

```
[AI的原话] → [推导过程] → [结论]

检查：
- 前提是否成立？
- 推导是否跳跃？
- 样本是否以偏概全？
```

### Step 4: 专家/工具验证（Expert Validation）

对于高风险内容，使用专项工具：

| 内容类型 | 验证工具/方法 |
|---------|--------------|
| 医疗信息 | PubMed、WHO官网、卫健委 |
| 法律条文 | 官方法规数据库、政府网站 |
| 统计数据 | 国家统计局、World Bank |
| 公司/产品 | 官网、Crunchbase、LinkedIn |
| 学术论文 | Google Scholar、Semantic Scholar |
| 代码/API | 官方文档、GitHub源码 |

## 高风险场景应对

### 场景A: AI提供了具体的数字/年份

```markdown
这个数字 [具体数字] 的来源是？
- 是否有原始报告/论文链接？
- 发布机构是否权威？
- 是否在知识截止日期之后？

验证方法：用 [数字+相关关键词] 搜索，确认是否能找到同一数据
```

### 场景B: AI引用了"研究表明..."

```markdown
请提供：
1. 论文标题/作者/期刊
2. DOI或链接
3. 研究年份和样本量

验证方法：在 Semantic Scholar (semanticscholar.org) 或 PubMed 搜索论文标题
```

### 场景C: AI涉及医疗/健康建议

```
⚠️ 警告：医疗信息幻觉可能造成伤害

必须验证：
- 建议是否符合WHO/卫健委指南？
- 是否有禁忌症或副作用被忽略？
- 剂量/疗程是否有依据？

建议用户咨询专业医生，不要仅依赖AI
```

### 场景D: AI给出了产品功能或价格

```
验证方法：
1. 官网查证（不要相信第三方"大全"类网站）
2. 在Reddit/Twitter搜索真实用户反馈
3. 直接联系官方客服确认
```

## Prompts to Guide Users

### 快速核查Prompt

```
你是一个事实核查员。请核查以下AI回答中的所有claims：
"[AI的回答]"

对于每个claim：
1. 标注置信度：[高/中/低/无法判断]
2. 标注风险：[高/中/低]（对用户决策的影响程度）
3. 给出验证建议：[需要查什么/去哪里查]
```

### 自我核查Prompt（让AI自己检查）

```
请检查你上一条回答中的以下内容：
1. 所有[数字/日期/统计]
2. 所有[引用/来源]
3. 所有[绝对化表述]

对于不确定的内容，请明确标注"需要验证"
```

## 预防策略（Grounding）

### 1. 结构调整Prompt

```
回答时，对于你无法100%确认的信息：
- 明确标注 [需要验证]
- 提供验证方向（去哪个网站/查什么关键词）
- 区分"你告诉我的事实"和"你的推测"
```

### 2. 来源优先Prompt

```
在回答前，先搜索你是否知道以下内容的可靠来源。
如果没有可靠来源，请明确说你不知道，而不是编造。
```

### 3. 不确定时的正确姿势

```
❌ 错误：编造一个听起来合理的答案
✅ 正确："这个问题我不确定，我的知识截止日期是[日期]，
         建议你去[具体网站/渠道]验证"
```

## Hallucination vs 错误

| 特征 | 幻觉（Hallucination） | 知识错误 |
|------|----------------------|---------|
| 表现 | 自信地编造不存在的内容 | 对真实事实的记忆偏差 |
| 来源 | 训练数据中的模式误用 | 知识库本身的不准确 |
| 特征 | 细节丰富但无法溯源 | 细节可能模糊或缺失 |
| 修正 | 需要外部知识纠正 | 可以通过提示修正 |

## Cost

完全免费（使用上述方法和公开工具）

## Limitations

- 无法实时验证2025年后的信息
- 某些专业领域需要真正的专家审核
- 工具核查依赖用户有网络访问能力
- 本技能不能替代专业领域的正式审查
